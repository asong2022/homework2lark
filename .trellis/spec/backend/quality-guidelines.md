# Backend Quality Guidelines

## Required Patterns

- Python 3.11-compatible type hints on public functions and ports.
- Pydantic v2 models validate every HTTP boundary and serialize camelCase API fields.
- SQLAlchemy 2.x `select()` style and explicit session/transaction ownership.
- Repository, `StorageAdapter`, and `OCRProvider` ports separate business rules from technology.
- Dependency injection allows Fake Provider and temporary storage/database in tests.
- Public timestamps are timezone-aware UTC; JSON and raw Provider output are serializable primitives.
- Image intake checks header dimensions against `MAX_IMAGE_PIXELS` before `ImageOps.exif_transpose()` or `load()`; a safety limit enforced after decode does not prevent decompression memory pressure.

## Forbidden Patterns

- `Any` at business boundaries, untyped dictionaries passed between layers, or blanket `# type: ignore`.
- Route functions containing SQL, image crop logic, or PaddleOCR parsing.
- Mutable default arguments, naive timestamps, silent exception swallowing, or secrets in source.
- Updating an existing OCR run/revision to represent a retry/edit.
- Declaring a problem reusable from OCR success alone.
- Adding Roadmap infrastructure without an active proposal/task.

## Tests Required by Change Type

- Domain rule/state change: unit test.
- Route or schema change: API integration test including error envelope.
- OCR/storage adapter change: port contract test.
- Migration change: fresh upgrade test plus ORM round trip.
- Lineage/normalized record change: full-flow round-trip assertion.
- Bug fix: regression test that fails before the fix.

The baseline suite must cover valid image upload, coordinate conversion/crop, successful and failed OCR, repeated OCR, repeated revisions, review gate rejection, reviewed eligibility, and normalized record retrieval.

## Tooling

- Ruff owns formatting and linting.
- Mypy runs in strict-enough mode for application source; do not disable whole modules.
- Pytest runs unit, contract, and integration suites.
- Coverage is diagnostic; acceptance-critical rules require named assertions, not only a percentage target.

### Windows Chinese-path runtime

This repository path contains Chinese characters. Keep `[tool.uv] package = false`: an editable `.pth` generated by uv can be UTF-8 while CPython 3.11 on Windows reads it with the local code page. Import source through the explicit project entry points instead:

- Pytest: `pythonpath = ["src"]`.
- Alembic: `prepend_sys_path = src`.
- Uvicorn: `uvicorn --app-dir src mistake_notebook_api.main:app`.
- Scripts: insert `apps/api/src` from `Path(__file__)`.

Wrong: re-enable editable installation and assume an English-path smoke test is sufficient. Correct: retain `package = false` until a Windows Chinese-path regression proves the `.pth` path is safe.

## Review Checklist

- Does data survive the full Storage → Repository → Service → API round trip?
- Are original, machine, and human layers still separate and immutable?
- Does each failure preserve previously committed evidence?
- Can the UI render a corrective message from the stable error code/message?
- Are full text, images, raw response, and secrets absent from logs?
- Does the change remain single-teacher/single-process compatible without claiming multi-user safety?
