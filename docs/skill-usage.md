# 智能错题学习 Skills：理念与使用说明

## 核心理念

这个系统要保存的不是一段 OCR 文本，也不是一张被裁下来的图片，而是一项经过教师确认、未来还能继续使用的“数学学习资产”。

一道完整题目包括题干文字、选择题选项、表格、线段图、几何图、统计图和其他必要视觉。它们共同决定题意，因此始终作为一道题保存。OCR、Yescan、MinerU 和 AI 只是帮助教师看清、提取和整理；最终题干、学生真实作答和教师诊断由教师确认。

本地存储和飞书 Base 分工不同：

- 本地证据库保存原图、裁图、OCR 原文、每次修订和匿名再练事件，保证可追溯且不覆盖。
- 飞书 Base 保存教师可读、Agent 可选择的题目目录、教学元数据、独立的变式题目录、题图和当前再练摘要，便于积累、筛选、组卷和复用。

因此，飞书 Base 是学习资产的中转核心，但不是唯一证据库。

`错题题目` 的 `收录序号` 是飞书自动维护的稳定顺序。误点其他列排序后，在列菜单中把 `收录序号` 设为升序即可回到最初的收录先后；编号不会因为视图排序而改变，常用视图也已默认按它升序。

## 一个总入口 + 按需专业能力

### `shi-homework2lark`

教师唯一需要记住的总入口。它负责：

- 接收图片、PDF、Word、单题截图和口述学情；
- 选择完整错题、OCR、教师确认修订和发布 Base；
- 从 Base 选题生成 `变式题1、变式题2……`；
- 把每道变式写入独立的 `变式题` 表，并关联回唯一原题；题目与可选答案解析分列，只有题目时也可直接复用；
- 先保留学生真实作答，再按同一道题、同一种错误原因汇总多名学生到 `错题记录`；
- 组织内容完整且未标记异常的原题/变式成为每日一练或练习卷；
- 记录匿名再练反馈并用于下一轮练习。

### 开始收集时先由你选择

只要本轮从“收集错题”开始，Hermes 必须先让你选，不能看完材料后替你决定：

1. **教师精选**：只发空白卷、整页材料或单题图，由教师选要保留的题；
2. **匿名批改统计**：空白卷 + 已批改作业，只统计题目、真实作答、人数和错误情况，不绑定姓名；
3. **实名绑定统计**：空白卷 + 已批改作业 + 私有名单，用学号和姓名核对，Base 默认只保存姓名。

你直接说“选 1”“匿名统计”或“按名单实名”都算本人选择。没有选择时，Agent 只能展示这三项，不调用 OCR、Web 或飞书。

内部按五个学习阶段工作，但它们不是五个需要教师记忆或分别安装的 Skill：

1. `收集入库`：接收材料、识别完整题目、教师确认后进入 Base；
2. `错情整理`：先保存真实作答，再汇总同题同因学生；
3. `题生变式`：从 Base 选原题，生成并写回独立的变式记录；
4. `精准练习`：按班级、学生或教师选择组题并生成 Word；
5. `再练反馈`：读取批次识别码和 manifest，把作答反馈关联回学习资产。

阶段之间使用 `[homework/…]` 交接块保存来源、问题、错情、Base、变式、练习和反馈状态。对话中断后，Agent 应从最后一个有效交接块继续，不重复 OCR、重复生成或重复写入。飞书、OCR、题图和文档生成仍由对应专业能力执行，总入口负责顺序、教师授权和数据血缘。

本机环境检查和标准整包验证：

```powershell
python -X utf8 .agents/skills/shi-homework2lark/scripts/doctor.py --strict
python -X utf8 .agents/skills/shi-homework2lark/scripts/package_bundle.py verify --package artifacts/skills/shi-homework2lark.skill
```

教师实际使用不需要运行这些命令；它们用于安装后自检或迁移到另一台 Codex/Hermes 电脑。

### `wumu-jihe-html`（“这道题画张图”）

可复用题图子技能。含图原题或变式需要重画题图时，由总入口调用它。它先恢复数学对象和约束，再生成离线、可编辑的 HTML；教师或 Agent 检查后导出 PNG。HTML 留在本地继续编辑，PNG 进入飞书和 Word。

## 最常用的对话方式

### 收集整页图片

```text
使用 shi-homework2lark。这里是一页作业，请在对话中列出完整题目候选。
我要保留第2题和第5题。第2题学生实际写的是……；我判断……。
先把题干、选项、题内图片、核心素养、设计意图和典型错例整理给我看，确认修订后存到飞书。
```

### 收集单题截图

```text
使用 shi-homework2lark。这张图本身就是一道完整错题，不用自动框题。
学生作答是……，我的诊断是……。先核对 OCR 和题图，确认后入 Base。
```

### 收集 PDF / Word

```text
使用 shi-homework2lark。请处理这份 PDF（或 Word），先保留逐页视觉，再让我选择错题。
题干文字、选项和题图必须作为同一道题；不要只根据提取出来的段落入库。
```

PDF 使用 MinerU 辅助理解并渲染逐页图片；Word 先渲染为 PDF/页面。当前本地 `/assets` 仍只接收图片，这个转换由 Agent 层完成。

### 空白卷 + 多份同版已批改作业

```text
使用 shi-homework2lark。这是一份空白卷和同版37份已批改作业。
空白卷每页只调用一次夸克 AI Agent 的 question-ocr，确认完整题目后复用模板区域；
不要给37份学生卷重复识别印刷题干。请结合学生真实作答、红笔批改和数学正误找错题，
先整理真实作答，再建议错因，相同错因的学生放在一行。现在先用AI看姓名学号，
以后我会给班级名单校验；飞书对应学生只保存姓名。
```

这条批量路径的重点是“题目只建一次，学生错误按证据分组”：

```text
空白卷 question-ocr/AI确认
→ 一次性完整题目与区域
→ 37份批改页对齐模板
→ 真实作答与批改语义
→ 错因建议与学生分组
→ 错题记录 + 人数/错误率/高频投影
```

如果需要夸克辅助识别身份，只把每页的“姓名 + 学号”小裁图加上页序号，排成少量高清联系表；不要把几十张整页作业缩在一张图里。教师提供名单后，以“学号 + 姓名”匹配结果为准。

如果作业有 2～4 页，不要一次把 44 人的 88～176 页塞进一个对话。先创建可续跑的录入活动，默认每批 16 页；一个学生的完整页集永不拆开：

```text
使用 shi-homework2lark。这份作业每人4页，请按名单开始分批录入，默认每批16页。
空白卷模板只识别一次；先加入学号01～04的完整4页，完成并保存进度后再继续下一批。
每批先记录真实作答和红笔证据，不要在观察结果里直接写错误原因或掌握状态。
```

Agent 使用 `staged_intake.py start/add/status/complete/export`。2 页作业通常每批 8 人，4 页作业通常每批 4 人；可在 4～24 页调整。活动接受教师指定的任意可读绝对路径，并把页面复制到私有活动目录；中断后用 `status` 查看进度。全对学生用空 findings 明确标记“已处理”，部分导出会清楚显示还剩多少学生和批次。

### 生成变式题

```text
使用 shi-homework2lark。从飞书“待生成变式”中读取尚未关联变式题的原题，默认生成3道。
题目和答案解析分开；答案解析可以留空。原题没有题干图片时默认只出文本；原题含图且新题仍依赖图时再配图。先预览，不要写回。
```

Skill 会先建立“年级 + 完整原题 + 数学本质 + 真实错例 + 错误表现 + 教师诊断 + 最近再练反馈”的事实卡，再使用内置的题生变式质量协议。默认贴近浙江小学数学期末评价常见风格：真实情境、合理数据、适龄语言和有效思考；没有提供真实样题时，不会假装参考过某一年的具体真题。

例如可以直接说：

```text
使用 shi-homework2lark。针对这道五年级“买几送几”错题生成3道变式。
学生的典型错误是没有把“买5送1”看成每6箱一组，请围绕这个关键关系设计。
题目贴近浙江五年级期末评价风格，但不要三道都只换数字；先给我看原题数学本质、每题变式点和质量检查，不要写回。
```

默认三题至少采用两个不同的主要变式轴，例如条件变化、问题反向、表征迁移或真实应用；所有题仍只叫 `变式题1、变式题2……`，不会增加分类列。每题即使不输出答案解析，Agent 也会先独立求解并检查条件、答案、年级、难度、单位和数学本质。

Skill 会区分完整裁图 `questionImageCount` 与题内视觉 `questionStemImageCount`，不会因为每道题都有完整截图就一律生成新题图。这个判断只是默认策略；教师明确要求“把文字关系改成线段图/几何图表达”时仍可配图。

每一道变式题占 `变式题` 表一行。每行保存题干、可选答案解析、设计意图和可选题图；核心素养自动引用原题。以后新增第 6、7 道不需要给原题表继续加列；`来源错题` 保持原题关系，`系统变式ID` 保证相同内容重复执行不会生成重复行。旧的同行 `变式题1～5` 已迁移并删除。

教师确认后说：

```text
写回。含图的第2题调用“这道题画张图”，图文核对并生成可读 PNG 后再一起保存；图片还没准备好就先不要写回。
```

### 汇总一组同类错误学生

```text
这道题里，学生甲、学生乙都把“70”看成整十数答案；学生丙是计算顺序错。
请先从这一组学生的真实作答中选一到几个典型错例，再建议错误原因；把相同错因的学生合并分组。
先给我看对应题目、题干图片、典型错例、错因建议、本组人数和对应学生。
我确认后再写入飞书“错题记录”，不要一个学生一行。
```

`对应学生` 是多选：可以直接使用姓名，也可以使用匿名代号。每一种共同错因一行；同题不同错因另起一行。`典型错例` 不写姓名，`题干图片` 直接引用原题。主字段使用 `日期｜题目名称｜错因摘要`，不会暴露学生姓名。个人错题集通过筛选 `对应学生` 得到，不创建“一个学生一张表”。人数显示为整数；错误率以一位小数百分比显示。批量作业还会把去重错误人数、错误率和错因人数汇总回题目行；只有错误率严格大于 35% 才标成统计高频。

### 学生个人精准练习

```text
请按我私有班级名单中的学号 01，为这名学生生成 7 月 15 日个人练习纸。
默认 6 题；从他真正关联的错题中选，优先需要再练的题，并尽量覆盖不同错误类型。
先放原错题，有余量再放这些错题对应的可用变式；不要用无关题凑数。
姓名和学号直接填进 Word，生成前告诉我实际能选几题。
```

Agent 会用名单把学号唯一核对到姓名，再通过 `错题记录.对应学生` 筛选该生错题；不会创建学生表，也不会把学号写进飞书。默认排除 `已掌握`，题目不足 6 道就按实际数量生成。个人 manifest 保存精确来源，右上角使用匿名实例码，例如 `20260715-01-S001-P1`；它不含姓名和学号。标题自动成为 `7月15日个人练习纸`，第一页自动填写姓名与学号。

真实命令由 Agent 执行：

```powershell
python <skill-root>/scripts/personal_practice.py schema-check
python <skill-root>/scripts/personal_practice.py plan --roster <私有班级名单.json> --student-number 01 --batch-code 20260715-01 --question-count 6 --output-dir <新目录>
python <skill-root>/scripts/practice_sheet.py --manifest <新目录>/manifest.json --output <新目录>/7月15日个人练习纸.docx
```

按完整班级名单批量生成时，直接说：

```text
请按这份私有班级名单，为全班批量生成7月15日个人练习纸，每人默认6题。
仍按每个人自己的错题选；没有合格错题的学生不要生成空白卷，在清单说明即可。
```

```powershell
python <skill-root>/scripts/class_practice.py schema-check
python <skill-root>/scripts/class_practice.py build --roster <私有班级名单.json> --batch-code 20260715-01 --question-count 6 --output-dir <新的班级批次目录>
```

也可重复传入 `--student-number` 只生成部分学生。匿名码始终来自完整名单顺序；只生成 03 号时仍是 `S003`。批次输出含教师 CSV 和逐生 manifest、选题报告、Word；文件名不含姓名学号。名单、批次目录和练习纸都是私有数据。批改扫描由 AI/OCR 读取页面码、题号、真实作答和红笔证据，再由 `retry_batch.py` 精确回到每生 manifest；一次正确默认只记为“练习中”，不会自动宣称已掌握。

### 组卷

```text
从飞书 Base 的“错题库”和独立“变式题/可组卷”视图选择原题或变式题，生成一份正式练习纸 Word。
学生卷不要答案；已有答案解析放在单独章节。某道题还没有答案解析时，可以按需生成或在教师卷中省略，不影响学生卷使用。含图题使用对应题图。
```

练习纸先冻结一份本地 manifest，再生成 Word。批次码使用 `YYYYMMDD-NN`，例如 `20260715-01`；标题自动成为 `7月15日练习纸`。重复页眉右上角文本框用批次前缀加 Word 当前页码字段显示 `20260715-01-P1、P2…`，页脚只显示页码，不再重复识别码，也不生成二维码。学生看到简洁连续题号，manifest 内仍以 `R01、R02…` 精确映射原题或变式。题干图片只在确实存在时放在对应题干文本下方并左对齐，随后才是作答空间；不按估算容量提前分页，Word 只有在当前页真正放不下下一道完整题目时才自然换页。识别码不含产品前缀、随机码、模板版本、学生姓名或飞书密钥。教师把批改后的纸张拍成扫描 PDF 后，AI/OCR 读取文字码和可见题号，系统自动定位 `Rxx → 原题/变式`、生成幂等反馈计划，并在既有授权范围内写回确定项；看不清的页单独列出，不影响同批其他题。

### 记录再练反馈

```text
孩子今天重做变式题2，实际作答是……。
我的判断是……。先给我看反馈摘要，确认后记录；不要保存姓名，也不要覆盖已有的典型错例和错误表现。
```

本地会追加一次包含真实观察的事件；飞书原题只更新 `掌握状态、最近再练时间`，实名错误组可更新 `再练反馈`。完整反馈不再重复塞进原题表。下一次生成变式时可读取本地事件或组级反馈，针对仍未解决的数学关系，而不是只换数字。

## 质量门

任何正式复用都遵守：

```text
OCR/AI草稿 ≠ 教师确认原题
写回但内容不完整的变式题 ≠ 可用练习
一次作答正确 ≠ 自动掌握
生成了图片 ≠ 图文一致
```

原题进入 Base 前仍需教师确认并保存非空修订；保存后不再执行第二次审核。Base 内不为目录整理、错题记录和完整变式重复建立审批列。写入、覆盖已有内容、生成 Word/打印/发送和反馈记录仍在对话中明确确认。原题的缺图、条件不完整、数学冲突、图文矛盾或身份无法匹配进入 `错题题目.需人工处理`；变式题输入或题图不完整时停止写入，不创建第二套复核列。

## 隐私

- 不要求学生账号或学生表。批量批改场景可用学号与姓名对照教师提供的班级名单；学号只用于本地核对，Base `对应学生` 默认只保存姓名。
- 使用 MinerU、Yescan、PaddleOCR-VL 等云服务前，Agent 应说明本次哪些材料会发送给第三方。
- 不把密钥、图片二进制、完整学生文本和远程附件 URL 写入日志。
- 如果材料含不必要的姓名或头像，优先裁切后再发送云服务。
