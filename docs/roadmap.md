# 路线图（Phase 1 之后）

本文件描述未来方向，不构成本次实现授权。任何条目进入开发前都需要独立 Trellis 任务；触发范围控制规则时先写 proposal。

## 产品主线

- 扫描、自动框题和 OCR 是可替换的录入路径，不是产品终点。教师上传整页、手动框题或直接提供单题截图，都应进入同一审核题目资产流程。
- 当前优先级已调整为：审核题资产 → 显式发布到飞书多维表格 → Codex/Hermes 从 Base 勾选题目 → 生成编号变式题 → 追加到独立 `变式题` 表并关联原题 → 后续组卷和重复使用。
- `shi-homework2lark` 是贯穿“多格式收集 → 完整题目审核发布 → Base 选题 → 变式/题图写回 → 组卷 → 再练反馈”的 Agent-first 总入口，不在 Web 后端固定接入某个 LLM API；AI、OCR 和题图输出都不能绕过教师审核。使用说明见 [skill-usage.md](skill-usage.md)。

## Phase 1B：教师复用试点（已批准）

- 已审核题目幂等发布到飞书 Base 的 `错题页面/错题题目`，教师通过短序号工作，隐藏系统 ID 保证幂等；本地数据库和文件仍是证据源。
- Web 入口以人工框题为主；对话入口可用 Yescan 整题候选；单题截图可直接按整图录入。三者汇入同一 OCR、修订、审核和发布流程。
- Base 勾选题目后，Codex/Hermes 默认生成 3 道编号变式；每道在独立 `变式题` 表占一行并关联原题，不再受五列上限限制。答案解析另列但不是复用前置条件。
- 所有生成统一称为变式题，可分别侧重四基巩固、表征变化、素养新角度、应用四能或挑战，不建立强制类别字段。
- 预览与写回是独立动作；题干完整且无真实异常的变式可直接进入组卷候选，缺答案解析不阻断学生卷。

## Phase 1C：个人/整班精准练习与回收（已批准）

- 使用教师私有班级名单按学号唯一核对姓名，不新增飞书学生表，不把学号回写 Base。
- 只读筛选 `错题记录.对应学生`，聚合到原题；默认 6 题，优先需再练和较新证据，并兼顾不同错误类型。
- 先选择该生真正错过的原题，有余量才轮换补入这些原题的可用变式；不足时不以无关题凑数。
- 生成 `M月D日个人练习纸` Word，自动填写姓名与学号；可见页码只加入匿名 `Sxxx`，身份映射保留在本地私有 manifest。
- 已支持按完整名单或学号子集批量生成，Word 自然分页且保留 `Sxxx/Rxx` 来源。
- 回收扫描由 Agent/OCR 观察，确定性脚本按页面码与题号定位 manifest、追加本地事件并准备 Base 投影；一次正确不自动宣称已掌握。
- 更长期的逐生纵向统计和正式学生/班级模型仍需独立任务。

## Phase 2：来源处理、检索与标签

- 多页 PDF 的显式逐页处理与 MinerU 文档提取（需独立 proposal/task）。
- 对话入口的一页多题候选质量改进；Web 自动候选只保留为可选辅助，不作为人工工作台前置条件。
- 图像增强、去手写等可选派生版本，原图仍不可覆盖。
- 题目视觉/文本去重与合并建议，不自动丢弃来源。
- 题库列表、检索、年级/学期/教材/单元/知识点标签。
- 教师审核式错误类型和错因标签。
- 审核题目目录、搜索、筛选与组卷候选集合；题目图片和文字题干作为同一资产读取。
- 提供单题截图直接录入通道，作为自动扫描质量不足时的可靠降级路径。
- 评估 MinerU、Quark Yescan、Doc2X 等第二 Provider；若接入多个，只先实现手动选择，自动路由/投票另行论证。

## Phase 3：错误事件、班级与掌握追踪

- 已先交付 Base 轻量切片：`错题记录` 按同题同类错误汇总多名学生标签，不创建学生表。
- 未来 `MistakeOccurrence`：仅在需要原子级逐学生历史、身份合并或正式统计时实现，一个 ReviewedProblem 对应多个学生/班级错误事实。
- 学生与班级最小身份/权限模型（需隐私、数据保留与协作 proposal）。
- 班级共性错题训练，以及个人练习从单生生成扩展到整班批量调度。
- `PracticeTask`、`PracticeAttempt`、作答证据、教师诊断和 `MasteryState`。
- 高频错题/掌握变化统计，但不把题目资产与错误事件合并。

## Phase 4：变式题深化与图形处理

在 Phase 1B 的同一编号变式题模型上，进一步提高生成质量和可控性。题目可以保持核心结构以巩固四基，也可以改变条件、问题、表征或路径；可以增加信息整合与迁移，也可以加入现实应用、策略比较、开放推理和适度挑战。系统不要求把每道题归入固定类别。

所有生成题进入正式题库/学生任务前必须通过：数学正确性、条件完整性、答案唯一/开放性、年级适切性、难度、图文一致性、与原题数学本质关系以及教师人工审核。

含图题可研究 `ImageGenerationProvider` 或 `DiagramProvider`，每个派生图必须保存来源图、Prompt/参数、Provider raw result、版本关系和人工审核；不得覆盖原题视觉证据。

## Phase 5：教学资源编排与效果分析

- 专题练习与个性化组卷。
- Word/PDF 导出与版式审核。
- 再练效果的跨批次趋势分析与长期教学干预评估。
- 巩固/变式/拓展/提升的组合策略。
- 再练效果、掌握稳定性和教学干预分析。
- 同年级教师协作、版本冲突与审核责任链。

## 未来题目扩展维度

ReviewedProblem 可关联：年级、学期、教材版本、单元、知识点、前置知识、数学核心素养、认知层级、问题情境、表征方式、难度、错误类型/原因、学生作答证据、教师诊断、变式关系、再练记录与掌握状态。

这些维度按真实用例逐步建模。Phase 1 不创建通用 EAV 表、向量库或知识图谱。

## 核心素养方向

未来分析与练习服务于：数学眼光（观察/发现）、数学思维（分析/推理）、数学语言（表达过程与结论），以及数感、量感、符号意识、运算能力、几何直观、空间观念、推理意识、数据意识、模型意识、应用意识和创新意识。

AI 只能提出候选标注/内容，不能绕过教师审核门槛。
