# 班级批量个人练习与分批录入 Proposal

## 出现的问题

现有能力可以为一名学生生成个人精准练习，但教师实际需要按完整班级名单一次生成多份不同内容的练习纸。另一方面，一次作业可能有 2～4 页；44 名学生会形成 88～176 页批改稿，若要求一次性上传和识别，容易超过对话附件、OCR Provider、上下文和人工核对的稳定边界，也难以在中断后继续。

## 为什么超出原切片

原个人练习切片明确只生成一名学生的一份练习；原错题录入也以一组来源材料完成一次流程为主。全班批量生成涉及输出目录、失败原子性、跨学生附件缓存和隐私清单；分批录入涉及一个作业活动跨多个处理批次的状态、页数预算、学生页完整性和可续跑导出，因此需要先记录边界再实现。

## 建议方案

### 班级批量个人练习

- 读取一份教师私有班级名单，默认遍历全部学生，也允许按学号选择子集。
- 对每名学生复用已经验证的个人选题规则：未掌握优先、原题优先、来源题覆盖、变式轮转和证据不足不凑数。
- 生成一个新的班级批次目录；每名学生使用完整名单中的固定匿名实例号 `Sxxx`，文件名不含真实姓名，Word 正文才填写姓名与学号。
- 产出私有批次摘要、教师清单、逐生 manifest、选题说明和 Word。没有可用错题的学生只进入清单，不生成空白练习纸。
- 全程只读 Base；任何技术错误使新批次整体失败并清理临时输出，不覆盖旧批次。

### 分批录入

- 一个录入活动只绑定一份空白卷模板和一份私有名单。空白卷每页的题目结构只需要识别一次。
- 教师把 PDF/Word 先按已有路由渲染为逐页 JPG/PNG；再按学生分批加入批改稿。
- 默认每批最多 16 页，可在 4～24 页内调整。系统永不把同一个学生的多页拆到两个批次：2 页作业通常每批 8 人，4 页作业通常每批 4 人。
- 每名学生必须提交与空白卷相同的连续页码集合；批次保存不可变来源副本、SHA-256、匿名实例号和处理状态。
- AI/OCR 只整理每名学生的真实作答、红笔证据和对应题号；先记录真实作答，再做错误归因。全对学生允许零条错题发现。
- 每批可独立完成；活动可随时查看进度并导出已完成结果。未完成时必须明确剩余人数和待处理批次，不得声称已覆盖全班。

## 数据模型影响

不修改本地数据库和飞书 Base 四表。新增的 `intake-campaign-v1`、批次清单和班级练习清单都是教师本地私有文件；后续仍由现有教师确认流程把整理好的错误分组写入 `错题记录`。名单、学号、学生页路径和逐生 manifest 不进入 Base 技术字段。

## 技术栈影响

继续使用 Python CLI、现有 Lark CLI 只读适配、Pillow 图片校验和 `python-docx` 组卷器。不引入任务队列、对象存储、学生账号、班级数据库或新的 OCR Provider。通过进程内缓存复用 Base schema、变式目录和题目附件，避免 44 名学生重复下载相同题图。

## 实施成本

中等。主要工作是抽取可复用的逐生选题服务、增加班级编排器、增加可续跑录入活动状态机，并补齐原子写入、隐私输出和批量测试。OCR/AI 识别本身继续由已有 Skill 对话流程负责。

## 是否建议进入下一阶段

建议立即作为当前教师试用主线的小切片实现，因为它解决真实 44 人班级的可操作性，但不扩大到学生账号、自动批改或班级分析。用户已于 2026-07-15 明确要求继续开发，视为对本 Proposal 的实施授权。
