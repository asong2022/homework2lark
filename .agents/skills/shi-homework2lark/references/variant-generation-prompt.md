# 题生变式生成协议

## 1. 适用范围

本协议用于：教师已确认原题修订并发布到飞书 Base，随后由 Codex、Hermes 或兼容 Agent 从 `错题题目` 选择原题，生成 1～5 道可独立使用的 `变式题1、变式题2……`。

它解决的是“怎样生成真正有教学价值、数学可靠的变式题”，不改变以下业务顺序：

```text
教师确认修订后的原题进入 Base
→ 教师/Agent 明确选题
→ 读取原题与可用学情证据
→ 生成并独立验算
→ 对话预览
→ 教师明确同意写回
→ 每道变式写入独立的变式题行
```

不得从未经教师确认的 OCR 草稿直接生成，不得虚构学生真实作答，也不得因为使用本提示词而新增“巩固 / 拓展 / 提升 / 挑战”等 Base 分类字段。

## 2. 生成前事实卡

先从 Base、题目图片和本轮对话中建立事实卡。未知值写“未知”或留空，不为填满模板而猜测。

```text
【任务】
- 原题稳定 ID：
- 生成数量：默认 3，允许 1～5
- 教师本轮意图：如“解决买几送几分组错误”“改变表征”“用于期末复习”
- 是否需要答案解析：可选；即使不输出，Agent 仍必须在内部独立求解
- 是否明确要求题图：

【教学定位】
- 年级：
- 学期：
- 教材版本：未知时不猜
- 地区/评价风格：默认“浙江小学数学期末评价常见风格”
- 单元/知识点：
- 认知与能力目标：

【原题事实】
- 完整题干：文字选项也放在这里
- 题内视觉：表格、线段图、几何图、统计图、数轴、图片选项等
- 标准答案/已有解析：可空
- 原题数学本质：核心概念、数量关系、图形关系或推理结构
- 原题关键条件与问题：
- 合理解法：

【学习证据】
- 典型错例：只使用已记录的真实作答；没有就留空
- 错误表现：可见事实
- 教师诊断/错误原因：可空，不把 AI 猜测说成教师判断
- 最近再练反馈：可空
```

年级可以从明确页眉、教材信息或教师说明中可靠取得。若年级未知且会实质影响知识边界、语言难度或数值范围，只生成带假设的预览并请教师确认，不能直接写回。

## 3. 可直接复用的正式提示词

把上面的事实卡填入后，使用以下提示词。不要把空字段自动补成虚构事实。

```text
你是一名小学数学命题组专家、学习评价设计者和一线教师。请依据现行义务教育数学课程标准所强调的核心素养与学业质量要求，基于下面这道已经由教师确认修订的原题，生成 {{生成数量}} 道编号变式题。

【目标】
生成的题目要帮助学生看见并迁移原题的数学本质，而不是只做表面换数。默认贴近浙江小学数学期末评价的常见风格：情境真实、语言简洁、数据合理、问题自然、认知负荷适合 {{年级}} 学生，并体现观察发现、分析推理、数学表达与解决真实问题的能力。

只有在上下文提供了真实试题样本时，才可以说“参考了该样题的命题思路”；没有样题时，不得声称检索或参考了某一年的具体真题。

【已知事实】
年级/学期：{{年级与学期}}
教材/地区信息：{{教材与地区；未知则写未知}}
完整原题：{{完整题干，含文字选项}}
题内视觉：{{视觉内容及数学关系；没有则写无}}
已有答案/解析：{{可空}}
原题数学本质：{{核心概念、数量关系、图形关系或推理结构}}
知识与能力目标：{{可空}}
典型错例：{{只放真实作答；没有则写无}}
错误表现：{{可空}}
教师诊断：{{可空}}
最近再练反馈：{{可空}}
教师本轮意图：{{可空}}

【命题过程】
1. 先在内部独立解出原题，核对题意、条件、单位和答案；若原题不完整、数学关系有歧义或无法可靠求解，停止生成并指出需要教师补充的最小信息。
2. 用一句话确认原题数学本质。变式可以改变数值、条件、问题方向、信息组织、表征方式或解题路径，但不得把核心考查对象换成无关知识。
3. 若存在真实错例、教师诊断或再练反馈，优先针对尚未掌握的关键关系设计；不得虚构学生经历、姓名、作答或错误原因。
4. 为每道题选择一个主要变式轴，并让整组题之间有实质差异。默认三题不能全部只是替换数字；可以包含较小变化、条件或问题变化、表征迁移、真实应用、多步骤推理、策略比较或解释说明，但输出仍统一叫“变式题N”，不要给它们增加“巩固/拓展/提升/挑战”分类。
5. 情境必须服务数学关系。不要把学校、超市、运动会等名词机械贴在纯计算题外面；数量、价格、时间、长度、人数和单位要符合常识，信息要必要且充分。
6. 语言适合该年级，避免成人化、学术化、歧义、过长阅读负担和超出课程范围的技巧。开放题必须说明可接受答案范围或评价标准；非开放题应有唯一确定答案。
7. 每道题生成后独立求解并交叉检查：数学正确、条件完整、答案确定或开放性明确、单位一致、数据合理、难度适切、与原题本质相关、与其他变式不重复。
8. 原题没有独立题干图片时默认生成可由文字完整表达的变式。只有新题确实依赖表格、几何图、数轴、统计图、线段图或图片选项，或教师明确要求改变表征时，才声明需要题图；题图必须与题干数量和关系逐项一致，并交给“这道题画张图”子技能制作。

【质量底线】
- 不照抄原题后只改人名或一个数字。
- 不用无意义情境装饰数学题。
- 不出现缺条件、多解却未说明、答案与题意不一致、单位冲突或年级越界。
- 不把“粗心”当作未经证据支持的错误原因。
- 不泄露学生姓名、学号或其他身份信息。
- 不展示长篇内部推理，只输出结论型设计依据、题目和校验结果。

【输出】
第一部分“生成依据”：用简短条目给出原题数学本质、本轮针对的学习证据、每道题的主要变式轴及整体难度是否适合，不输出冗长思维过程。

第二部分“教师预览”：依次输出“变式题1、变式题2……”。每题必须包含完整题干和简明设计意图；文本选项属于题干。答案解析按教师要求提供，可以留空，但你仍须先完成内部验算。确需题图时，说明图中必须出现的对象、数量和关系，不得假装图片已经生成。

第三部分“质量检查”：逐题用通过/需修改报告数学正确性、条件完整性、答案确定性或开放性、年级适切性、难度、单位/数据、与原题本质关系以及图文一致性。任何一项“需修改”都先重写该题，不进入写回 payload。

第四部分“写回草稿”：只在全部检查通过后输出符合约定的 JSON；此时仍是只读预览，等待教师明确说“写回”。不要添加 category、level、type 或“巩固/拓展/提升”字段。
```

## 4. 变式轴选择

变式轴是 Agent 的设计工具，不是 Base 分类。每道题选择一个主要轴，必要时辅以一个次要轴：

| 变式轴 | 设计方式 | 防止退化 |
| --- | --- | --- |
| 数值与约束 | 改变数据范围、整除关系、单位或必要约束 | 不能只换数字而保持所有思考完全相同 |
| 条件变化 | 增减、重组或隐含一个需要提取的条件 | 条件必须必要且充分，不人为制造阅读噪音 |
| 问题变化 | 由求结果改为求条件、比较方案或反向推理 | 不得改变到与原题数学本质无关 |
| 表征变化 | 文字、表格、线段图、数轴、几何图之间转换 | 只有真正需要时才制作题图 |
| 信息组织 | 从顺序给出改为筛选、匹配或跨句整合 | 阅读负荷仍需适合年级 |
| 解题路径 | 允许多种方法、策略比较或解释理由 | 必须给出清晰评价标准 |
| 应用迁移 | 将同一关系放入可信的新情境 | 情境中的数量和行为必须真实可行 |
| 再练定向 | 针对真实错例或最近反馈改变易错触发点 | 不复述学生身份，不把推测当事实 |

默认生成 3 道时，至少使用 2 个不同的主要变式轴。教师若明确只想做基础同构练习，可以覆盖这一默认值。

## 5. 强制质量门

Agent 在对话预览前完成下面的检查。失败项直接重写，不把未通过内容写入 Base：

1. **原题可解**：已独立解出原题，原题本身没有影响结论的缺失信息。
2. **本质一致**：可以用一句话说明新题与原题共享的数学本质。
3. **数学正确**：重新代入或用另一种方法验算，答案与题意相符。
4. **条件完整**：没有缺条件、冲突条件、无用噪音或未经说明的多解。
5. **年级适切**：知识范围、语言、数值、步骤和抽象程度适合指定年级。
6. **真实自然**：情境和数量符合常识，不为了“素养”强行编故事。
7. **差异有效**：整组题不是同一道题的同义改写；默认三题至少两个主要变式轴。
8. **证据诚实**：真实错例、教师诊断和 AI 推测清楚分层，不虚构学生事实。
9. **媒介必要**：文本能完整表达就不额外造图；需要图时图中对象、数量、标注和题干逐项一致。
10. **隐私安全**：题干、设计意图和 payload 不包含学生姓名、学号或可识别信息。

## 6. 写回 JSON 契约

`variant_catalog.py` 只接收持久化所需字段。每道题的设计意图写入同一条变式记录；整组质量检查保留在对话预览和本地任务上下文，不再为 Agent 名称或生成备注增加 Base 列。

```json
{
  "questionId": "problem_xxx",
  "variants": [
    {
      "question": "完整变式题题干，文字选项也在这里",
      "answerAnalysis": "可选：答案、关键过程和与原题数学本质的关系",
      "designIntent": "说明改变了什么、为什么这样改，以及希望学生形成的数学思考"
    },
    {
      "question": "另一道完整变式题题干",
      "designIntent": "通过表征变化促进学生理解数量关系",
      "diagram": {
        "required": true,
        "description": "题图必须呈现的对象、数量、标注与数学关系",
        "localPng": "session/diagrams/problem_xxx-v2.png",
        "editableHtml": "session/diagrams/problem_xxx-v2.html"
      }
    }
  ]
}
```

`answerAnalysis` 可以省略或为空，`designIntent` 必填。声明 `diagram.required=true` 时，必须先按 [Diagram contract](diagram-contract.md) 调用 `wumu-jihe-html`，完成 HTML、PNG 和图文核对；图片未就绪时不得执行写回。检查通过的 PNG 直接附在同一变式题行，不再维护题图说明或变式复核列。

## 7. 停止并向教师确认的情况

- 原题图片或题干缺失，无法判断完整条件。
- 原题本身有歧义、答案不唯一但未说明开放性，或已有答案疑似错误。
- 年级未知且不同年级会改变知识边界、语言、数值或解法要求。
- 教师意图与已记录学情冲突，例如要求回避而又要求专门训练同一关键关系。
- 含图题的关键标注看不清，或图形关系会影响答案。

确认前可以给出“基于某项明确假设的草案”，但不得执行 `variant_catalog.py write`。
