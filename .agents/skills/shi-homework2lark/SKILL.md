# shi-homework2lark · 智能错题学习工作流

这是教师与 AI 共建可持续数学学习资产的**唯一入口**。它负责调度、阶段门禁和交接，不复制飞书、OCR、题图或文档 Skill 的领域实现。

架构采用“单入口 + 厚 reference + 确定性 script”：五段流程共享同一题目谱系、Base 契约和练习批次，不能拆成五个会抢触发的子 Skill。

## 激活后先做

1. 首次安装、首次绑定飞书、教师说“帮我配置/初始化”或环境不确定时，先完整读取 [一键安装与配置](references/setup-guide.md)，运行 `onboarding.py check`，并且每次只向教师索取当前唯一缺失项。不得先讲开发测试。
2. 若同级存在 `../shi-shared/SKILL.md`，先完整读取并遵守；缺失时使用本 Skill 内的隐私、费用和边界规则，不伪装已经读过。
3. 始终完整读取 [编排与交接合约](references/orchestration-and-handoffs.md)，从上下文填写最小 `[homework/task]`，不要重复询问教师已经给出的信息。
4. 若本轮从收集错题开始，先读取 [用户三选一收集方式](references/collection-modes.md)。没有用户明确选择时只展示 1/2/3 并等待，禁止 AI 自动判断或调用任何外部能力；选择后运行 `workflow.py start` 固化 `decidedBy=user`。
5. 判断本轮从哪个学习阶段开始、停在哪里。教师说“一口气做完”可连续推进，但外部上传、Base 写入、覆盖和反馈更新仍执行对应授权门。

只检查 Skill 包自身时可运行离线体检；它不联网、不认证、不读密钥：

```powershell
python <skill-root>/scripts/doctor.py
```

完整首次配置使用同一确定性入口：

```powershell
python <skill-root>/scripts/onboarding.py --repo-root <repo-root> check
```

脚本会依次检查仓库运行时、PaddleOCR、飞书用户授权和空白 Base，只返回一个下一步。内置 `.base` 是快速创建载体，`base-schema.json` 与 `base-views.json` 是验证和升级契约；教师不需要手工建表或理解字段 ID。

## 五个学习阶段

| 阶段 | 用户常见说法 | 本轮主结果 |
|---|---|---|
| A 收集入库 | “把这些错题收进去”“这张卷选几题” | 完整题目、人工修订证据、Base 原题 |
| B 错情整理 | “分析这批批改作业”“哪些学生同一种错” | 真实作答优先的同题同错因分组与 Base `错题记录` |
| C 题生变式 | “给这道题举一反三”“生成变式题” | 编号变式、可选题图、写回独立 `变式题` 行 |
| D 精准练习 | “每日一练”“给全班/某学生出卷” | 不可变 manifest、班级或个人 Word 练习纸 |
| E 再练反馈 | “这是练习后的批改结果”“更新掌握情况” | 观察事实、教师判断、反馈事件与 Base 当前投影 |

允许从任意已完成阶段继续，不强迫重新收题。已有可靠 `[homework/…]` 交接块时先复用并验证，不从头猜测。

## 正确业务顺序

```text
材料/空白卷
→ 完整题目候选
→ 教师选择
→ OCR/AI 整理
→ 教师确认修订
→ Base 原题
→ 已批改作业中的真实作答
→ 同题同错因分组
→ Base 错题记录
→ Base 选题生成变式
→ 原题/变式组卷
→ 再练反馈
→ 下一轮针对性学习
```

扫描只是输入路径。不得先生成变式再整理原题，也不得把未经教师确认的 OCR/AI 输出直接当作正式题目或学生事实。

## 收集方式不是 AI 路由结果

收集开始前必须由用户选择：

1. 教师精选；
2. 匿名批改统计；
3. 实名绑定统计。

如果用户已明确说“我自己选题”“不实名”或“按名单实名”，这就是用户选择；否则只展示三项并停止。AI 不得说“我判断适合第 2 种”后直接执行。选择完成后，Agent 自动推进该方式所需的材料检查、OCR/看图、对话内核对、Base 入库和阶段交接。

## 不可破坏的规则

1. 一道题是一个逻辑资产。题干文字、文字选项、表格、线段图、几何图、统计图和图片选项共同组成一题，不按 OCR block 拆题。
2. 外层印刷题号只写 `题号`，不重复进入 `题干文本`；只在已知题号与开头标签完全匹配时剥离，不能误删题干本身的数量。
3. 原材料、OCR 原文、AI 建议、人工修订、Base 投影和生成内容分层保存，后一步不覆盖前一步。
4. 原题进入 Base 前必须存在教师确认后的非空人工修订；保存该修订后即可发布，不再创建另一套“待审核/已审核”状态。只有原题真实缺失、矛盾或身份冲突进入 `错题题目.需人工处理`。独立变式题也不设第二套复核字段，输入或题图不完整时直接停止写入。
5. `错题记录` 一行表示同一道题、同一次作业、同一种错误原因的一组学生；先保存典型真实作答和可见错误表现，再提出原因。不要每名学生一行，也不要创建学生表。
6. 飞书 Base 是教师与 Agent 的长期目录；本地存储保留原图、裁图、OCR、修订、manifest 和事件历史。Base 不能替代本地证据层。
7. 私有名单保存学号与姓名映射；Base `对应学生` 只写名单中的准确姓名，选项顺序按学号排列。学号保留在私有名单、manifest 和 Word，不进入稳定 ID 或普通日志；出现同名学生时必须停下并另行设计消歧方案。
8. `错题题目.核心素养` 是唯一可编辑事实源，页面、错题记录和变式题只查找引用；原题与变式题的 `设计意图` 分别说明收录价值和变式目的。AI 生成的变式先独立求解和质量检查。所有题统一叫 `变式题N`，不新增“巩固/拓展/提升/挑战”等分类列；默认三题至少使用两个不同的主要变式轴，题图只在新题确实依赖视觉条件时生成。
9. 个人/班级练习只使用真实关联的错题及其可用变式，不用无关题凑数。Word 使用自然流式分页，不插入预估硬分页符。
10. 多页批改作业按页数预算分批，但一个学生的完整页集不拆批；空白模板每页的印刷题只识别一次。
11. 外部上传和写入遵循最小授权。教师可一次授权一段明确流程；失败时保留已完成证据，不静默扩大范围或自动多 Provider 投票。

进入外部 Skill 前必须完整读取它的 `SKILL.md` 及其要求的 reference。若文件系统没有 `lark-base`，但 `lark-cli` 已安装，则先运行 `lark-cli skills read lark-base` 读取随 CLI 同步发布的官方说明，再按它要求读取相关 reference；这属于同一官方能力，不得凭记忆猜参数。其他外部 Skill 找不到时报告受影响阶段和可用降级路径。

题生变式遇到原题不完整、年级会改变适切性、数学关系有歧义或题图关键条件不清时，只能给出带假设预览并向教师补最小信息，不能执行写回。

## 调度纪律

1. **先选路再执行**：先展示简短任务卡和本轮终点；能从上下文确定的字段不再追问。
2. **交接可见**：每完成一阶段，更新对应 `[homework/…]` 块和短进度表；私有长文本、名单和远端 ID 只留在私有文件。
3. **事实先于归因**：学生真实作答、红笔批改证据和数学正误先落地，原因与掌握状态后置。
4. **一个写入门**：同一明确范围可以一次授权连续写入；范围、记录集合或覆盖语义改变时重新确认。预览/doctor/schema-check 不算写入。
5. **可停止、可续跑**：教师可停在任一阶段。保存已有交接块、稳定 ID 和私有产物位置，下次从该处继续。
6. **失败不倒退**：OCR、Base、附件或 Word 失败时保留已经完成的不可变证据，只重试失败步骤。
7. **Web 只是框题器**：Agent 对话是主入口；需要可视化题目边界时才打开现有 SourceAsset，可手动或显式自动框题。点击完成后 Web 只返回公开 `problemId`，OCR、整理、质量门和 Base 写入继续由 Agent 执行。

## 最小授权门

| 动作 | 是否需要明确授权 |
|---|---|
| 本地只读分析、doctor、schema-check、预览 | 否 |
| 把含学生信息的完整材料发送第三方 OCR | 是；说明发送给哪一家 |
| 保存教师已经确认的题目修订 | 收录请求已明确时直接执行；覆盖既有修订需单独确认 |
| 发布原题、写错题分组、写变式、更新反馈到 Base | 是；可授权明确范围内的连续动作 |
| 创建本地 Word/manifest | 用户已明确要求生成时直接执行 |
| 覆盖、删除或批量修改既有 Base/文件 | 必须单独确认 |

## 交付方式

完成时先报告教师真正得到的结果：收录了多少题、形成多少错误组、生成多少变式/练习、哪些学生无合格题、哪些阶段未做。随后给私有产物位置、Base 变化摘要和下一步建议；不要用一串技术命令代替结果。

## 失败与范围边界

- 不默认并行调用多家 OCR，不自动投票，不静默从低价路径升级高价路径。
- 不创建学生账号、学生表、班级权限、向量库、消息队列、微服务或后台自动整库生成。
- 返回练习扫描件由 Agent/`shi-ocr` 观察，`retry_batch.py` 按页面码和题号自动定位 manifest、生成幂等事件及 Base 投影计划；不确定项不自动写入。
- “全自动”不等于绕过教师对题目内容的确认、第三方上传和外部写入授权，也不等于仅凭一次正确自动标记已掌握。
- 题图数学关系有歧义时先确认；图片生成成功不等于图文正确。
- PDF/Word 页面化失败时保留原件并请求 PDF/逐页图片，不得声称已经识别。
