# 五木几何编辑器

## 核心原则

- 先还原数学关系，再计算坐标和绘图，不按原图像素描摹。
- 只交付纯白背景、完全离线的单文件 HTML。
- 直接调用生成脚本复用内置模板；执行任务时不要预读或解析模板源码。

## 工作流

1. 从题目文字和图片提取对象、定义、约束与依赖；歧义影响数学关系时再询问用户。
2. 用解析公式或确定性几何计算派生点、交点、中点、垂足等坐标；识图时恢复完整数学结构，不照搬断线、缺弧和透视误差。需要 OCR 纠错或立体图判断时读取 `references/geometry-rules.md`。
3. 创建临时 JSON：顶层只含 `elements` 数组；每项包含 `tag`、`kind`、`attrs`，文字另加 `text`。普通对象使用 SVG 标签；圆柱、圆锥必须使用 `tag: "solid"` 及 `cx/cy1/cy2/rx/ry`，正方体、长方体必须使用 `tag: "solid"` 及 `cx/cy/w/h/ox/oy`，由脚本生成可调整体与规范虚实线，不得拆成独立线弧。按图层顺序输出：阴影或填充 → 主体轮廓 → 点、标记与标签。识别构造关联，把必须随主体移动的点、标签、边和图形设置相同 `group`；可独立调整的辅助线、尺寸线和注释不纳入该组。
4. 从图形核心内容提炼不超过 10 个汉字的文件名摘要，按本地生成时间命名为 `<摘要>-YYMMDD-HHMMSS.html`，年月日、时分秒均取两位数，例如 `等腰梯形内的两半圆-260713-191533.html`。运行 `python3 scripts/generate_svg_editor.py <数据.json> <命名后的文件.html>`；脚本会把 SVG 对象插入模板预留位置。
5. 若本地预览可用，打开 HTML 并至少截图核对一次图形、标签、阴影、布局和编辑状态；发现问题先修正再复查。预览不可用时说明原因并做静态检查，最后删除临时 JSON。
6. 交付生成的本地 HTML。

## 与智能错题协作

由 `shi-homework2lark` 调用时，题图仍属于同一道原题或同编号变式题，不创建新题目。除 HTML 外，打开编辑器检查后使用右上角“导出 PNG”；HTML 作为本地可编辑源，PNG 作为飞书/Word 题图。返回对应的题目稳定 ID、变式编号、HTML/PNG 相对路径和简短数学约束说明。导出前必须逐项核对题干中的数量、标签、角度、平行/垂直、相等与包含关系；不得把答案或解题辅助线误画进学生题图。
