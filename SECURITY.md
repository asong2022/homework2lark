# 安全与隐私政策

## 报告安全问题

请优先使用 GitHub 的私有漏洞报告功能，不要在公开 Issue 中提交以下内容：

- API Key、Token、Cookie、Base token 或授权截图；
- 真实学生姓名、学号、班级、头像或作答图片；
- 本地数据库、`.env`、日志、OCR 原始响应或第三方附件 URL；
- 可定位个人设备、目录或账号的绝对路径。

报告中请提供最小复现步骤、受影响版本和不含敏感数据的合成样例。

## 密钥泄露处理

如果密钥曾出现在提交、日志或聊天记录中：

1. 立即在服务商后台撤销或轮换密钥；
2. 停止相关自动任务；
3. 检查远端仓库、Release、CI 日志和构建产物；
4. 再清理 Git 历史。仅删除当前文件不能使旧密钥恢复安全。

## 学生数据

学生材料默认属于私有数据。真实姓名和学号只应在教师控制的本地名单中核对；公开仓库、示例和测试必须使用合成数据。启用云端 OCR 或题目检测前，应确认材料范围和服务商，并尽量只发送教师已经选定的题目裁图。

## 支持范围

安全修复以默认分支的最新版本为准。Phase 1 不提供完整权限系统、多租户隔离或合规托管承诺；部署者需要自行保护运行主机、飞书账号和第三方 Provider 凭据。
